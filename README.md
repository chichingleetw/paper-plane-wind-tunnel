# 紙飛機風洞實驗室

一個可以直接部署到 GitHub Pages 的單頁 3D 教學小遊戲。玩家可以在瀏覽器中調整紙飛機的風速、攻角、側風、翼寬與重心，觀察彩色氣流、升力、阻力、重力與飛行軌跡如何一起影響紙飛機的表現。

這個作品的核心不是追求專業 CFD 精度，而是把空氣動力學中最重要的關係變成可操作、可觀察、可比較的互動實驗。

## 線上展示

部署完成後可透過 GitHub Pages 開啟：

```text
https://chichingleetw.github.io/paper-plane-wind-tunnel/
```

## 專案特色

- 使用 Three.js 建立 3D 紙飛機、風洞場景、氣流粒子與受力箭頭。
- 提供風洞觀察與自由飛行兩種模式。
- 可調整風速、攻角、側風、翼寬、重心前移與投擲速度。
- 以速度色階呈現氣流變化：藍色代表較慢，紅色代表較快。
- 即時顯示相對空速、升力、阻力、高度、飛行距離與滯空時間。
- 攻角過大時會顯示失速與氣流分離提示。
- 不需建置流程，單一 `index.html` 即可執行與部署。

## 原理說明

### 1. 相對空速

紙飛機真正感受到的是「相對於空氣的速度」，不是單純的地面速度。自由飛行模式中，程式會把飛機速度與風速向量做比較，得到相對空速。這會直接影響升力與阻力。

### 2. 升力

升力使用簡化公式估算：

```text
L = 1/2 * rho * v^2 * S * Cl
```

其中 `rho` 是空氣密度，`v` 是相對空速，`S` 是翼面積，`Cl` 是升力係數。遊戲中可以看到：風速或投擲速度提高時，升力會因為速度平方項而明顯增加；翼寬增加時，有效翼面積也會變大。

### 3. 攻角與失速

攻角是紙飛機機翼與來流方向之間的角度。攻角增加時，升力通常會先上升；但超過約 15 度後，氣流開始分離，升力不再穩定增加，阻力會快速變大，這就是失速現象。

在遊戲中，當攻角過大時，氣流粒子會變得混亂，右側也會顯示失速警告。

### 4. 阻力

阻力由基本阻力、升力造成的誘導阻力，以及高攻角額外阻力組成。這能讓玩家看到一個重要取捨：想要更多升力時，往往也會付出更多阻力，飛機可能更快減速。

### 5. 重心

重心前移會讓紙飛機更穩定，但也可能需要更高速度或更合適的攻角來維持滑翔。遊戲中以「機頭重量／重心前移」滑桿模擬這個效果，讓玩家可以比較不同重心配置對飛行距離與滯空時間的影響。

## 應用情境

- **STEM 教學**：用互動方式說明升力、阻力、攻角、失速與相對風。
- **科展與課堂展示**：讓學生先在模擬器中調參，再摺紙飛機實測比較。
- **遊戲化學習**：把抽象公式轉成任務式挑戰，例如「最遠距離」或「最久滯空」。
- **設計思考練習**：比較滑翔型、飛鏢型、經典型紙飛機在不同條件下的表現。
- **網頁 3D 範例**：展示如何用單一 HTML 檔整合 Three.js、UI 控制與簡化物理。

## 小遊戲玩法

### 風洞觀察模式

1. 在「模式」選擇 **風洞觀察**。
2. 調整風速、攻角、側風、翼寬與重心。
3. 觀察紙飛機周圍的彩色氣流粒子。
4. 查看右側的升力、阻力與相對空速數值。
5. 嘗試把攻角調高到 15 度以上，觀察失速警告與尾流變化。

適合用來理解單一參數如何影響氣流與受力。

### 自由飛行模式

1. 在「模式」選擇 **自由飛行**。
2. 選擇紙飛機類型：滑翔型、飛鏢型或經典型。
3. 設定攻角、風速、側風、翼寬、重心與投擲速度。
4. 按下 **投擲／開始**。
5. 觀察飛行軌跡、飛行距離、滯空時間、高度與速度變化。
6. 每次只改變一個條件，再重新投擲，比較結果差異。

挑戰目標：

- 找出最遠飛行距離的參數組合。
- 找出最長滯空時間的參數組合。
- 在有側風時，讓紙飛機仍保持穩定航線。
- 避免攻角過大造成失速。

## 操作方式

- 滑鼠拖曳：旋轉視角。
- 滾輪：縮放視角。
- `投擲／開始`：在風洞模式啟動觀察，在自由飛行模式投擲紙飛機。
- `暫停`：暫停或繼續模擬。
- `慢動作 x0.25`：放慢模擬速度，方便觀察。
- `重置`：回到初始位置與狀態。
- 顯示選項可開關氣流粒子、受力箭頭與飛行軌跡。

## 本機執行

因為專案使用 ES module import map 載入 Three.js，建議用本機伺服器開啟：

```bash
python3 -m http.server 8000
```

然後在瀏覽器開啟：

```text
http://localhost:8000
```

## 部署到 GitHub Pages

1. 將專案推送到 GitHub repository。
2. 到 GitHub repository 的 **Settings → Pages**。
3. Source 選擇 **Deploy from a branch**。
4. Branch 選擇 `main`，資料夾選擇 `/root`。
5. 儲存後等待 GitHub Pages 建置完成。

本專案沒有額外 build step，GitHub Pages 會直接提供根目錄的 `index.html`。
